Displaying View Data

If the Database has any Views setup, then they are displayed under the Views tab.


Views are customized subset of Tables data. They are very similar to Tables except that

  • They can combine data from more than one Table.
  • They are usually Read-only. Even if they can be Updated, DbSee displays them as Read-Only.
  • They need to be setup in the Database, using an SQL statement, usually by an authorized person such as a Database Administrator.


Displaying View data

Selecting a View in the Views Tab, displays Columns of the selected View at the bottom panel titled View Columns. You can optionally Select or Unselect the columns before opening the View.


Like Tables, Views can be opened in two different ways, to display their data:

  • If the number of records that will be displayed is small (less than a few thousand) just Check the check box to the left of the View to Open it.
  • If the number of records that will be displayed is Large then click on the button. On the Filter Dialog that is displayed select the columns and set the conditions and click OK.


Both Query Parameters and Open Filtered are used to filter the View data. The Open Filtered method is more flexible:

  • You can select and set filter on any number of Fields. Query Parameters have fixed set of filters built-in and the conditions (comparison operator) cannot be changed.
  • Open Filter dialog is more user friendly with drop down Calendar for date fields.


If you use DbSee primarily for extracting data you will benefit from setting up Views and executing them with Open Filtered method to get the exact data you want.
