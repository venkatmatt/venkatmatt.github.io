Viewing Table Data

To display a Table's data simply click the checkbox to the right of the Table Name on the Tables Tree List. The data is displayed on the right under a tab whose caption will be the Table's name.

Trying to open a Table with a large number of records without any filtering will consume memory resources, slowing down the system, sometimes making it appear locked up! See Filtering Data for more information on Filters.

If you have set the Show Filter Dialog before Opening a Table or View on in DbSee Options then a Filter Dialog will be displayed whenever you attempt to open a Table or View.


Data Display

Table or View Data is displayed in a Data Grid and can be manipulated and controlled as illustrated in Data Grid Features.

  • You can open more than one Table (or View or Query) at a time. Each Table's data is displayed under a separate tab and you can quickly switch between them by clicking on the tab. You can also select any tab using the Tabs  Menu displayed to the right of all tabs.
  • You can close the table by clicking the cross mark on the tab or unchecking it in the Tables Tree List.
  • The currently selected Record (row on the Data Grid) can also be viewed in a Record View which is a grid with two columns Column Name and Value. The Record View Panel located to the right of the Data Grid is usually hidden - click the button on the Grid Toolbar to toggle its visibility. The Record View is also used to display Many-to-One Lookups, illustrated in Quick Demo - Record View & Lookup.
  • If the Table (or View) has Memo, Blob fields then they are displayed separately in a panel below the Data Grid as illustrated in Memo & Blob View.
  • If the Table has self-referencing Foreign Keys (i.e a Foreign Key that refers to the Primary Key of the same table) then you can display its data in a Tree-Grid instead of a normal Grid, if you have checked the Show Tables with Self Referencing Foreign Keys in a Tree Grid option under Data Tab of Options Dialog.

A self-referencing Foreign Key appears with the icon (yellow instead of green used for normal Foreign Keys) in the Columns List displayed below the Tables List.


Master-Detail Data Browsing

With Interbase, MSSQL and Access Databases DbSee can extract Primary and Foreign Key information and using this info it will display the Tables in a Table Tree depicting the Master-Detail Relations, up to a depth of 2 - i.e Master ¢ Detail ¢ Sub-Detail. See Quick Demo Master-Detail for an illustration of Master-Detail Data Browsing.

Note that a Table, in addition to appearing in Root Node as Master, may also appear in Child Node(s) as Detail, when it has a Master-Detail relation with another Master. Table appearing in Child nodes can be opened only if their parent Master Table is already opened - when the parent Master Table is not open, the child table's check box is grayed out.


When you open a Master Table appearing in Root node

  • It is always opened under a separate Tab in the Data Display area.
  • It displays all its data, unless you used a Filter Dialog before opening to filter data.


Opening a Detail Table, appearing in a Child Node

  • Displays its data in a Detail or Sub-Detail Panel under the same tab as the Master Table.
  • Displays only the data that corresponds to the currently selected record in its Master.
  • You cannot use a Filter Dialog before opening the Detail Table (the Open with Filter button is disabled for Detail Tables), as it is already filtered for its Master's record. You can however apply filtering to the data displayed, using the Auto Filter features of the Grid.


Many to One Lookup

When a Table has Many-to-One relation with another Table, you can Lookup the Master record of the Other Table corresponding to the current record of the Table you are browsing. For e.g while viewing the data of a Products table, you can lookup the record from Supplier table, that corresponds to the SupplierID of the current record of Products. The looked-up data is always a single record and is displayed under a tab in the Record View, described above. See Record View & Lookup under Quick Demo for an illustration of the Lookup feature.
