Creating And Editing Queries

When DbSee starts, the Query tab is initialized for a New Query File containing a New Query with a skeleton SQL Statement Select * From. You can also start a New Queries File by clicking the Create a New Queries File button.

  • At the Bottom panel enter/edit the Name (Description) for the Query, and press Enter
  • At the SQL Editor below enter the SQL Statement for the query.
    • You can float the SQL Editor Panel by clicking the Float button. Floating Editor has larger width and is resizable so more convenient for creating a new query. You can dock back the floating editor by clicking on its Dock button.
    • The Query Editor has syntax highlighting.
    • You can also quickly insert SQL Keywords. Enter the first one or two characters and press Ctrl Space-bar - at the popup displayed select the Keyword you want to insert.
  • Test the Query by clicking the Execute button. Edit and Test again till done.
  • You can add more queries to the current file. Click the Add a New Query button to add a blank query. Follow the steps above to complete it.
  • Finally Save the queries to a file by clicking the Save As (or Save) button. Enter a suitable File Name for the Queries file in the Save Dialog and click OK. By default queries files are saved with the .SQF file extension - if you want you can specify a different extension when you enter the file name.


If you already have a set of queries for a database, you can organize them into a single file. Create a New Query File as Described above and cut and paste the SQL Statemnt one by one, adding a suitable Name to each and finally saving the file.

An SQL Query can have one or more optional Query Parameters that are used to filter the data when the query is executed. When a query with parameter(s) is executed, DbSee displays a Enter Query Parameter dialog - Enter a value for each of the parameters (you cannot leave them blank) and click OK.

For query parameters use the : prefix instead of the ? , for e.g

instead of
select * from ORDERS where ShipCountry = ?

use
select * from ORDERS where ShipCountry = :ShipCountry

The identifier that follows the : (ShipCountry in the example) above is used the prompt for that parameter in the Enter Query Parameter dialog.
